Detailed Description

Simple DNS cache of hostname to IP address with TTL. A full pass through the cache is required for add and lookup so this is not intended for high volumes, not that you'd have the SRAM for that anyway. A pessimistic average domain name length of 25 characters with 20 entries would use around (26*20)+(12*20) = 760 bytes of SRAM in the cache. Whatever, parsing 20 entries per lookup is massively cheaper than a network DNS lookup.
