Detailed Description

template<class TConnection, class TUser = void>
class stm32plus::net::TcpServer< TConnection, TUser >

Class to manage a listening TCP server. Each TCP server has a port on which it listens for incoming connections. When a remote client attempts a SYN with the server we create a connection object to handle the interaction and leave it to handle the state machine.

Unlike the common berkeley sockets implementation there is no concept of a 'listen' backlog because the API is entirely event based. The server is created in an unstarted state. The user should subscribe to accept events and then call start().

Template Parameters:
TConnectionThe user's derivation of the library-supplied connection class.
TUseroptional type of a pointer that the user would like to be passed to the connection constructor.
