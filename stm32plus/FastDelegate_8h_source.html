00365 // On MSVC and Intel, there isn't enough information in the pointer itself to
00366 // enable conversion to a closure pointer. Earlier versions of this code didn't
00367 // work for all cases, and generated a compile-time error instead.
00368 // But a very clever hack invented by John M. Dlugosz solves this problem.
00369 // My code is somewhat different to his: I have no asm code, and I make no 
00370 // assumptions about the calling convention that is used.

00372 // In VC++ and ICL, a virtual_inheritance member pointer 
00373 // is internally defined as:
00374 struct MicrosoftVirtualMFP {
00375   void (GenericClass::*codeptr)(); // points to the actual member function
00376   int delta;    // #bytes to be added to the 'this' pointer
00377   int vtable_index; // or 0 if no virtual inheritance
00378 };
00379 // The CRUCIAL feature of Microsoft/Intel MFPs which we exploit is that the
00380 // m_codeptr member is *always* called, regardless of the values of the other
00381 // members. (This is *not* true for other compilers, eg GCC, which obtain the
00382 // function address from the vtable if a virtual function is being called).
00383 // Dlugosz's trick is to make the codeptr point to a probe function which
00384 // returns the 'this' pointer that was used.

00387 // It has a trival member function that returns the value of the 'this' pointer.
00388 struct GenericVirtualClass : virtual public GenericClass
00389 {
00390   typedef GenericVirtualClass * (GenericVirtualClass::*ProbePtrType)();
00391   GenericVirtualClass * GetThis() { return this; }
00392 };

00525 // DelegateMemento -- an opaque structure which can hold an arbitary delegate.
00526 // It knows nothing about the calling convention or number of arguments used by
00527 // the function pointed to.
00528 // It supplies comparison operators so that it can be stored in STL collections.
00529 // It cannot be set to anything other than null, nor invoked directly: 
00530 //   it must be converted to a specific delegate.
00531 
00532 // Implementation:
00533 // There are two possible implementations: the Safe method and the Evil method.
00534 //        DelegateMemento - Safe version
00535 //
00536 // This implementation is standard-compliant, but a bit tricky.
00537 // A static function pointer is stored inside the class. 
00538 // Here are the valid values:
00539 // +-- Static pointer --+--pThis --+-- pMemFunc-+-- Meaning------+
00540 // |   0        |  0       |   0        | Empty          |
00541 // |   !=0              |(dontcare)|  Invoker   | Static function|
